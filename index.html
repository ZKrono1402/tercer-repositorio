Formato básico de Estructura de Commit:

  • type: El tipo de cambio (elegir uno de la lista).
  • scope: El área o módulo afectado (opcional, pero útil, como "animations" o "packaging").
  • short sumary: Un resumen corto en tiempo presente, sin punto al final (ejemplo: "corrige un error" en lugar de "corrigió un error.").

Ejemplos de Tipos de Commits

  • feat: Nueva funcionalidad para el usuario.
    • Ejemplo: feat(login): agrega botón de inicio de sesión

      Descripción: Añade un nuevo botón en la página de login para que los usuarios puedan iniciar sesión de manera más fácil.

  • fix: Corrección de errores para el usuario.
    • Ejemplo: fix(profile): arregla el crash al cargar datos

      Descripción: Corrige un problema donde la página de perfil fallaba al intentar mostrar los datos del usuario.

  • docs: Cambios en la documentación.
    • Ejemplo: docs(readme): actualiza instrucciones de instalación

      Descripción: Actualiza el archivo README con pasos más claros para instalar el proyecto.

  • style: Cambios de formato (sin afectar código).
    • Ejemplo: style(css): ajusta espaciado en el formulario

      Descripción: Modifica el CSS para mejorar el espacio entre los campos del formulario, sin cambiar su funcionalidad.

  • refactor: Mejora del código (sin nueva funcionalidad).
    • Ejemplo: refactor(utils): renombra función a algo más claro

      Descripción: Cambia el nombre de una función en las utilidades a uno más descriptivo para facilitar el mantenimiento.

  • test: Agregar o mejorar pruebas.
    • Ejemplo: test(auth): añade prueba para validar contraseña

      Descripción: Agrega una prueba automática para verificar que las contraseñas cumplan con las reglas establecidas.

  • chore: Cambios en tareas o configuración.
    • Ejemplo: chore(build): actualiza versión de Node.js

      Descripción: Actualiza la versión de Node.js requerida en el archivo de configuración del proyecto.
